# 夜间行人检测系统 v1.1.0 使用说明

## 实时检测

1. 模型、设备、`imgsz`、置信度和IoU位于顶部公共控制区。
2. `auto`设备会优先选择CUDA GPU，不可用时自动使用CPU。
3. 普通RGB视频或摄像头使用3通道模型；配对RGB/IR视频或双摄像头使用6通道模型。
4. 切换模型、设备、性能模式或跟踪选项时，正在运行的视频会安全停止并重新开始。
5. 源码入口会自动使用MKL顺序线程层，避免Anaconda环境中的NumPy/MKL与PyTorch重复加载Intel OpenMP。
6. 使用PyCharm和Conda解释器时，在app运行配置的环境变量中设置`MKL_THREADING_LAYER=SEQUENTIAL`，使该值在Python启动前生效；本机项目配置已自动补齐。

## 跟踪与计数

- 开启“跟踪”后使用ByteTrack生成稳定行人ID。
- “当前”是本帧行人数，“累计”是当前视频会话出现过的不同ID数量。
- 每次开始新视频或摄像头会重置ID统计，已收集的导出记录不会自动清空。

## 警戒区域

1. 点击“设置警戒区”。
2. 在右下角Result画面中按住鼠标拖出矩形。
3. 行人中心点首次进入区域时发出声音并记录事件。
4. 事件目录包含同名JPG截图、MP4录像和JSON元数据。
5. “清除警戒区”会关闭区域判断，但不会删除已经保存的文件。

## 双模态同步

- `IR帧偏移 > 0`：视频开始时跳过对应数量的IR帧。
- `IR帧偏移 < 0`：视频开始时跳过对应数量的RGB帧。
- 水平/垂直偏移以像素为单位，在红外预处理和6通道拼接前应用。
- RGB+IR实时双摄采用先同时`grab`、再分别`retrieve`的方式减小采集时间差。

## 性能模式

- 精度：每帧使用完整NLM和CLAHE，适合图片分析或GPU。
- 均衡：每帧使用实时NLM和CLAHE，兼顾速度与输入一致性。
- 流畅：6通道视频隔帧执行模型推理，中间帧复用跟踪框，并禁止跳读源视频。

## 记录与诊断

- CSV/JSON记录包括时间、来源、帧号、行人ID、置信度、坐标、警戒状态、人数、设备和FPS。
- “设备信息”显示软件版本、操作系统、CPU、PyTorch、OpenCV、CUDA、GPU显存和模型通道。
- 源码模式设置保存在`runs/gui_config/settings.json`；EXE模式设置和结果位于`%LOCALAPPDATA%\NightPedRGBT`。
